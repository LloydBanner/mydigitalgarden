## Ten Commandments for good Referring
- Ideas for how an OSR DM should play
- Be happy to make stuff up when needed, don't always play as written 
- learn rules
- Don't act against the players specifically but don't favour them either
- Be consistent with decisions
- Describe combats more cinematically 
- Let story happen organically and allow players to try anything, basically don't railroad

## Dynamism and the Generic Optimum
- Creat dynamic encounters by breaking up the normal order of play (e.g split up party, or reverse healing and damage)
- Boss fights are often dynamic because normal tactics can't always be used but don't use them all the time
- Examples, water puts out torches, enemies run for reinforcements 
- If players can still use the same tactics it is not dynamic 
- How to do this, Attack all of character sheet, Introduce in the middle of combat, not always magic
